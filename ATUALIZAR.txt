Sim. Como seu deploy ficou em /var/www/apicontratos/ContratosN/ com as pastas API/ e FRONTEND, o fluxo de atualização daqui para frente deve ser por git pull + rebuild dos containers.

O modelo certo é este:

Fluxo normal

Você altera localmente.
Faz git add, commit e push para o GitHub.
No Ubuntu, entra na pasta do projeto.
Faz git pull na pasta que mudou.
Rebuilda e sobe de novo com Docker.
Se a alteração for só na API:
cd /var/www/apicontratos/ContratosN/API
git pull origin main
docker compose up -d --build api
docker compose logs --tail=200 -f api

cd /var/www/apicontratos/ContratosN/FRONTEND
git pull origin main

cd /var/www/apicontratos/ContratosN/API
docker compose up -d --build web
docker compose logs --tail=200 -f web

Se mudou API e frontend juntos, faça os dois git pull e depois:

cd /var/www/apicontratos/ContratosN/API
docker compose up -d --build
